Stage 01 · Foundation
Data
Engineering
Ingest, store, version, validate, and transform raw data into ML-ready datasets

The bedrock of every ML project. Raw data from databases, APIs, streams, and files is ingested into a data lake, transformed via pipelines, validated for quality, and versioned for reproducibility. Nothing downstream works without this being solid.

4 AWS · 5 OSS
AWS Services
S3
Amazon S3
Scalable object storage — your data lake foundation. Store raw data, processed datasets, model artifacts. Virtually unlimited, ~$0.023/GB/month.
Managed · Pay-as-you-go
GL
AWS Glue
Serverless ETL — discover, catalog, and transform data. Glue Catalog acts as a Hive-compatible metadata store. Auto-generates ETL code.
Managed · Serverless
KN
Amazon Kinesis
Managed streaming data ingestion for real-time ML pipelines. Kinesis Data Streams for ingestion; Firehose for delivery to S3/Redshift.
Managed · Per-shard pricing
AT
Amazon Athena
Serverless SQL on S3. Query your data lake without loading into a database. Pay per query. Integrates natively with Glue Data Catalog.
Managed · $5/TB scanned
Open Source
SP
Apache Spark (on EMR)
Distributed data processing at any scale. Run on AWS EMR Serverless for massive transformations without cluster management.
Apache 2.0
dbt
dbt Core
Transform data in your warehouse using SQL + Jinja. Version-controlled data models, lineage tracking, tests. Runs free against Athena/Redshift.
Apache 2.0
DL
Delta Lake / Apache Iceberg
ACID transactions on S3. Table format that adds versioning, time-travel, and schema evolution to your data lake. Essential for reliable ML datasets.
Apache 2.0
GE
Great Expectations
Define data quality rules as code. Validate datasets automatically in pipelines. Catches schema drift and anomalies before they corrupt training data.
Apache 2.0
DV
DVC (Data Version Control)
Git for datasets and ML artifacts. Track large files in S3 with a git-like CLI. Full lineage from raw data → trained model. Reproducible experiments.
Apache 2.0
Stage 02 · Features
Feature
Store
Centralize, reuse, and consistently serve engineered features for training and inference

The feature store bridges data engineering and model training. It eliminates training-serving skew by guaranteeing that the same feature logic runs in both environments. Teams share features instead of recomputing them, dramatically reducing duplication.

1 AWS · 2 OSS
AWS Services
SF
SageMaker Feature Store
Fully managed online (DynamoDB-backed, low-latency) and offline (S3-backed) feature store. Point-in-time correct feature retrieval for training. Integrates with SageMaker Pipelines natively.
Managed · Pay-as-you-go
Open Source
FS
Feast
The leading open-source feature store. Define features in Python, deploy to Redis (online) + S3/Parquet (offline). Point-in-time joins for training datasets. Works perfectly with AWS infrastructure.
Apache 2.0
HP
Hopsworks (Community)
Open-source platform with a mature feature store, feature pipelines, and a model registry. Community edition is self-hostable on EC2 or EKS. Python-native API.
Apache 2.0 (Community)
Stage 03 · Research
Experi­mentation
& Tracking
Log runs, compare metrics, tune hyperparameters, reproduce results

Where data scientists live. Experiment tracking captures every run — parameters, metrics, code version, artifacts — so any result can be reproduced. Hyperparameter optimization automates the search for the best configuration. The IDE keeps everything collaborative.

2 AWS · 4 OSS
AWS Services
MF
SageMaker MLflow (Managed)
Fully managed MLflow tracking server on AWS. No infrastructure to spin up — just a tracking URI. IAM auth, S3-backed artifact store, persists across sessions. The easiest way to run MLflow on AWS.
Managed · Hourly pricing
ST
SageMaker Studio
Browser-based IDE purpose-built for ML. Manage experiments, notebooks, training jobs, and model deployment from one unified UI. Integrates with all SageMaker services.
Managed · Free IDE, pay for compute
Open Source
ML
MLflow (self-hosted)
The open-source experiment tracking, model registry, and lightweight serving framework. Self-host on EC2 with S3 artifact store + RDS backend. Or use SageMaker's managed version above.
Apache 2.0
OP
Optuna
State-of-the-art hyperparameter optimization framework. Define a search space in Python, Optuna finds the best config using TPE, CMA-ES, or grid search. Integrates with MLflow for logging.
MIT
JU
JupyterLab
The standard interactive notebook environment. Run in SageMaker Studio or on EC2. Extensions for Git, MLflow, and visualization. Where exploration begins.
BSD 3-Clause
ZN
ZenML
Open-source MLOps framework that unifies pipelines with experiment tracking. Abstracts infrastructure — the same pipeline code runs locally or on SageMaker Pipelines.
Apache 2.0
Stage 04 · Core
Training
& Compute
Scale model training from a single GPU to massive distributed clusters

The computational heart of MLOps. AWS provides fully managed training infrastructure that auto-provisions GPUs, handles distributed training, and integrates with your experiment tracker. Open-source frameworks provide the actual model training logic.

3 AWS · 5 OSS
AWS Services
TR
SageMaker Training Jobs
Fully managed distributed training — submit a job, AWS provisions GPU/CPU instances (p4d, g5, Trn1), runs training, stores artifacts to S3, then terminates. Pay only for training duration.
Managed · Per-second billing
PP
SageMaker Pipelines
Native CI/CD for ML — define training pipelines as DAGs. Rerun on schedule or on data triggers. Steps include processing, training, evaluation, model registration. Full lineage tracking.
Managed · Free orchestration, pay for compute
ES
EMR Serverless
Run Spark ML workloads at scale without managing clusters. Submit PySpark jobs for large-scale feature engineering or batch inference. Automatically scales to zero when idle.
Managed · Per vCPU-hour
Open Source (Frameworks)
PT
PyTorch
The dominant deep learning framework. Native AWS SageMaker containers available. Supports DDP for distributed training. PyTorch Lightning for structured training loops.
BSD 3-Clause
HF
Hugging Face Transformers
Pre-trained models (BERT, LLaMA, Mistral) and training utilities. AWS has official SageMaker HuggingFace containers. Fine-tune foundation models on your own data.
Apache 2.0
SK
scikit-learn
The gold standard for classical ML — regression, classification, clustering, preprocessing. Integrates perfectly with SageMaker's sklearn container. Lightweight and dependency-free.
BSD 3-Clause
XG
XGBoost / LightGBM
Gradient boosted trees — often outperform deep learning on tabular data. AWS SageMaker has a built-in optimized XGBoost container. LightGBM trains faster on large datasets.
Apache 2.0
RY
Ray Train
Distributed training orchestrator for PyTorch/TF. Scale from laptop to cluster with one line change. Handles worker coordination, fault tolerance, and checkpointing on EC2.
Apache 2.0
Stage 05 · Governance
Model
Registry
Version, stage, and govern models — the gatekeeper before production

The model registry is the single source of truth for production-ready models. It stores versioned model artifacts, tracks metadata (training data, metrics, author), enforces staging gates (Staging → Production), and integrates with CI/CD for automated deployment triggers.

1 AWS · 2 OSS
AWS Services
MR
SageMaker Model Registry
AWS-native model versioning with approval workflows. Register models from training jobs, attach metadata, require human approval before promotion to production. Triggers EventBridge on state changes for automated CI/CD.
Managed · No additional cost
Open Source
MR
MLflow Model Registry
Version models logged during experiments. Transition through Staging → Production → Archived with comments and lineage. REST API for downstream integration. Works whether self-hosted or on SageMaker managed MLflow.
Apache 2.0
BN
BentoML
Package models (any framework) into a standardized Bento format with dependencies, preprocessing, and serving logic baked in. The Bento is your deployment artifact. Integrates with MLflow.
Apache 2.0
Stage 06 · Production
Deployment
& Serving
Expose models as APIs, streaming endpoints, or batch jobs with low latency at scale

Deployment is where models create value. Real-time endpoints serve predictions in milliseconds. Batch transform handles millions of records asynchronously. CI/CD ensures safe rollouts with canary testing and rollback. Infrastructure as code makes it all repeatable.

4 AWS · 4 OSS
AWS Services
EP
SageMaker Real-Time Endpoints
Managed REST API endpoints — deploy any model container, get a HTTPS URL. Auto-scaling, multi-model endpoints, shadow testing, canary deployments. Supports data capture for monitoring.
Managed · Per instance-hour
BT
SageMaker Batch Transform
Async batch inference on large datasets stored in S3. Provisions instances, runs predictions, writes output, terminates. No persistent endpoint needed — cost-efficient for non-real-time use cases.
Managed · Pay per job
CD
AWS CodePipeline + CodeBuild
CI/CD for ML model deployment. Trigger from model approval in registry → build container → run integration tests → deploy to endpoint. Integrates with GitHub via CodeConnections.
Managed · Pay-as-you-go
LC
AWS Lambda + API Gateway
Serverless ML inference for lightweight models. Zero cold-start for small models, sub-millisecond latency. Ideal for feature preprocessing, post-processing, or routing between models.
Managed · Per request
Open Source
FA
FastAPI
High-performance Python API framework for custom model serving. Build your own inference server with auto-generated OpenAPI docs. Containerize with Docker, deploy to SageMaker or ECS/EKS.
MIT
TN
Triton Inference Server
NVIDIA's open-source inference server — optimized for GPU workloads. Supports PyTorch, TensorFlow, ONNX, TensorRT simultaneously. Use as SageMaker container for high-throughput serving.
BSD 3-Clause
TF
Terraform / CDK
Infrastructure as code for your entire MLOps stack on AWS. Define SageMaker endpoints, S3 buckets, IAM roles, and pipelines reproducibly. AWS CDK uses Python/TypeScript for native AWS abstractions.
MIT / Apache 2.0
GH
GitHub Actions
Free CI/CD for public repos (unlimited) or 2,000 min/month for private. Trigger model training on PR merge, run DVC pipelines, push Docker images to ECR, deploy to SageMaker endpoints.
Free tier / MIT runner
Stage 07 · Sustain
Monitoring
& Observ.
Detect drift, alert on degradation, close the feedback loop, trigger retraining

Models degrade silently — the world changes but the model doesn't. Monitoring catches data drift, prediction drift, and infrastructure failures. When degradation is detected, automated retraining pipelines can kick off Stage 3–5 all over again, closing the loop.

2 AWS · 4 OSS
AWS Services
MM
SageMaker Model Monitor
Continuously monitors real-time endpoints for data quality, model quality, bias, and feature attribution drift. Captures input/output data to S3, runs scheduled analysis jobs, triggers CloudWatch alarms on violations.
Managed · Pay per monitoring job
CW
Amazon CloudWatch
AWS-native metrics, logs, dashboards, and alarms. Monitor endpoint latency, error rates, invocation counts. Set alarms that trigger EventBridge → Step Functions → retraining pipeline automatically.
Managed · Pay per metric/log
Open Source
EV
Evidently AI
Open-source ML observability — generate drift reports, data quality checks, and model performance tests as HTML or JSON. Run in SageMaker Processing Jobs on a schedule. Integrates with Grafana dashboards.
Apache 2.0
PR
Prometheus + Grafana
The standard OSS observability stack. Scrape metrics from SageMaker endpoints, Kubernetes pods, and custom model servers. Grafana dashboards visualize latency, throughput, drift scores, and business KPIs.
Apache 2.0
AH
Apache Airflow (MWAA)
Schedule retraining pipelines triggered by monitoring alerts. A DAG watches for drift → pulls fresh data → kicks off SageMaker Training Job → registers new model. AWS MWAA provides managed Airflow.
Apache 2.0
NB
Nannyml
Estimates model performance without ground truth labels — critical when you don't get labels for days or weeks after prediction. Detects covariate shift and CBPE performance estimation on live data.
Apache 2.0
